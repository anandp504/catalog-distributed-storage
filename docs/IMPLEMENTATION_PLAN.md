## Phase 1: Consistent Hash Ring

**Goal:** Given a `catalogId`, deterministically return the URL of the Gitea node that should own it. This is the **initial-placement-only** policy — it is never consulted for catalogIds that already have a Postgres record.

### Files created/modified

- `src/coordinator/hashRing.js` (implement)
- `tests/unit/coordinator/hashRing.test.js` (write first)

### TDD: Failing tests first

**File:** `tests/unit/coordinator/hashRing.test.js`

| Test case | Asserts | Starts failing because |
|-----------|---------|----------------------|
| Single node always returns that node | `ring.getNode('X') === nodes[0]` for any X | `hashRing.js` is a stub |
| Two nodes split 1000 IDs each within 20–80% | counts[node] in [200, 800] | no implementation |
| Same ID always maps to same node (100 calls) | all calls return identical URL | no implementation |
| Adding a node changes ≤ 1/(N+1) assignments | transition fraction ≤ ~25% for 3→4 nodes | consistent hash property not implemented |
| Wrap-around: ID hashes beyond last vnode | still returns a valid node (not undefined) | binary search without wrap missing |
| Empty node list throws | `new HashRing([])` throws `'HashRing requires at least one node'` | no guard |
| Virtual node count: 150 per node | ring length === `nodeUrls.length × 150` | hardcoded or missing |
| Recreating ring with same nodes → identical routing | `ring1.getNode(id) === ring2.getNode(id)` for all IDs | non-deterministic if not sorted |

### Implementation notes

- Ring: sorted array of `{ hash, nodeUrl }` — `hash` is `sha256('${nodeUrl}:${i}')` for `i` in `[0, virtualNodes)`
- Lookup: binary search for first entry with `hash >= sha256(catalogId)`; if none, wrap to index 0
- Export: `class HashRing { constructor(nodeUrls, virtualNodes = 150); getNode(catalogId): string }`

```js
// key pattern
const keyHash = crypto.createHash('sha256').update(repoId).digest('hex')
// binary search → first ring[i].hash >= keyHash; wrap to 0 if none found
```

## Phase 2: Routing Table (Postgres + Redis)

**Goal:** Three-layer routing resolution: (1) Redis cache hit → return node, (2) Redis miss + Postgres hit → return node + populate Redis, (3) both miss → call hash ring, INSERT Postgres, HSET Redis, return node. Also implements `sanitizeCatalogId`.

### Files created/modified

- `src/coordinator/routingTable.js` (implement)
- `tests/unit/coordinator/routingTable.test.js` (write first)

### TDD: Failing tests first

Uses `jest.mock('pg')` and `jest.mock('ioredis')` (or `ioredis-mock`) — no live database in unit tests.

**File:** `tests/unit/coordinator/routingTable.test.js`

| Test case | Asserts | Starts failing because |
|-----------|---------|----------------------|
| Redis cache hit → returns node, zero Postgres calls | `pgQuery` not called | `routingTable.js` is a stub |
| Redis miss + Postgres hit → returns node, populates Redis | `redis.hset` called with correct args | no implementation |
| Redis miss + Postgres miss → calls hash ring, INSERTs Postgres, HGETs Redis | all three side-effects fire in order | no implementation |
| `ON CONFLICT DO NOTHING` — two concurrent first-publishes → same node | second INSERT doesn't overwrite first Postgres row | not implemented |
| Adding a new node to ring does not change existing Postgres entries | old catalogId still routes to original node after ring expansion | stateless lookup — rings don't mutate Postgres |
| `sanitizeCatalogId('../foo')` throws | error thrown | not implemented |
| `sanitizeCatalogId('/etc/passwd')` throws | error thrown | not implemented |
| `sanitizeCatalogId('foo\\bar')` throws | error thrown | not implemented |
| `sanitizeCatalogId('foo\0bar')` throws (null byte) | error thrown | not implemented |
| `sanitizeCatalogId('CAT-GROCERY-001')` returns the string | no throw, returns input | not implemented |
| `routingTable.close()` closes Postgres pool and Redis | no error on cleanup | not implemented |

### Implementation notes

- Postgres: `pg.Pool` from `DATABASE_URL`; query: `SELECT node_url FROM catalog_routing WHERE catalog_id = $1`
- Postgres write: `INSERT INTO catalog_routing (catalog_id, node_url) VALUES ($1, $2) ON CONFLICT DO NOTHING`
- Redis: `ioredis` client from `REDIS_URL`; key: `HGET catalog-routing {catalogId}` / `HSET catalog-routing {catalogId} {nodeUrl}`
- Write order: Postgres first, Redis second — never Redis-only
- `sanitizeCatalogId`: reject if matches `/[\/\\]|\.\.|\x00/`; return validated string or throw with `{ code: 'INVALID_CATALOG_ID' }`
- Export: `class RoutingTable { constructor(pgPool, redisClient, hashRing); resolve(catalogId): Promise<string>; close(): Promise<void> }`

```js
// resolve() outline
const cached = await redis.hget('catalog-routing', catalogId)
if (cached) return cached
const { rows } = await pg.query('SELECT node_url FROM catalog_routing WHERE catalog_id = $1', [catalogId])
if (rows.length) {
  await redis.hset('catalog-routing', catalogId, rows[0].node_url)
  return rows[0].node_url
}
const nodeUrl = hashRing.getNode(catalogId)
await pg.query('INSERT INTO catalog_routing (catalog_id, node_url) VALUES ($1, $2) ON CONFLICT DO NOTHING', [catalogId, nodeUrl])
await redis.hset('catalog-routing', catalogId, nodeUrl)
return nodeUrl
```

### Implementation notes

- `readFile(catalogId, filePath)`: GET contents; base64-decode `data.content`; parse JSON; return null on 404; throw on other non-2xx
- Never swallow HTTP errors silently — always throw with `${method} ${url} → HTTP ${status}`

### Implementation notes

**`src/common/becknShapes.js`:**
```js

const nack = (errorCode, errorMessage) => ({ status: 'NACK', error: { errorCode, errorMessage } })

```

- `validatePublishRequest(body)` → `null` or `{ errorCode, errorMessage }`

## Hard Rules (enforced in every phase)

- **No git library on client** — `node-fetch` only; no `simple-git`, `isomorphic-git`, `child_process git`
- **Postgres before Redis** — every routing write goes to Postgres first, then Redis; never Redis-only
- **Redis is cache-only** — `--save ""` in Docker; a flush must never lose routing data
- **`sanitizeCatalogId` is the single source of truth** — defined in `routingTable.js`, imported everywhere
- **NACK shape** — always `{ status: 'NACK', error: { errorCode, errorMessage } }`; use `becknShapes.nack()`
- **Node URLs from env** — `GITEA_NODES` only; never from request body
